# Operating Workflow

## Project Flow

`idea → planning → doing → testing → completed → archived`

- **Idea:** problem and expected value are captured.
- **Planning:** scope, success metric, architecture, milestones, and risks exist.
- **Doing:** the current milestone has executable tasks.
- **Testing:** acceptance criteria, evaluation, and release checks are running.
- **Completed:** outcome and retrospective are documented.
- **Archived:** no active work; context remains searchable.

## Task Contract

Write tasks where the work context lives:

```markdown
- [ ] Implement inference health check #project/elysia ⏫ 📅 2026-07-10
```

Use one owner, one clear verb, one outcome, and a due date only when commitment exists.

## Cadence

- **Daily:** choose three outcomes, process inbox, close the loop.
- **Weekly:** review projects, deadlines, blocked tasks, and learning.
- **Monthly:** review goals, portfolio, systems, and what to stop.

## Kanban Policy

Each project board uses: Ideas, Planning, Doing, Testing, Completed. Limit Doing to three cards per project.
